DIAGNOSTIC ULTRATHINK
─────────────────────
`Dashboard.tsx:loadData` wrappait **5 queries Firestore** dans un seul
`try/catch` global (ligne 75-176). Si UNE seule query throw, tout cassait
avec bannière rouge "Erreur lors du chargement du tableau de bord".

Inventaire des 5 queries (avec analyse de risque) :
| # | Query                                                | Risk | Catch ? |
|---|------------------------------------------------------|------|---------|
| 1 | `getDocs(collection(db, 'quotes'))`                  | low  | NON     |
| 2 | `Promise.all([commissions, notes_commission])`       | low  | OUI(.catch=>null pré-V46) |
| 3 | `getDocs(query(sav, where statut=='nouveau'))`       | MED  | NON     |
| 4 | `getDocs(query(containers, where statut in [...]))`  | HIGH | NON     |
| 5 | Tri local recents (pas Firestore)                    | low  | N/A     |

Coupables probables :
- Query 4 (`containers`) : le `where in [...]` peut nécessiter un index
  Firestore composite. Si pas indexé → throw `FAILED_PRECONDITION`.
  Plus probable : la collection `containers` n'existe pas ou est vide.
- Query 3 (`sav`) : collection `sav` peut être absente.

Note importante (à monitorer) : Dashboard interroge `containers` (anglais),
V49 RBAC rules matchent `/conteneurs/` (français). Le fallback admin-only
des rules couvre `containers` (admin a accès), donc pas un blocker. Mais
les 2 conventions FR/EN coexistent — à harmoniser en V51.

FIX V50-BIS — Promise.allSettled granulaire
───────────────────────────────────────────
Refactor en 4 helpers async indépendants :
   - `loadQuotesStats()`     : devis + CA + recents
   - `loadCommissionsStats()`: commissions dues + liste
   - `loadSavCount()`        : nb SAV nouveau
   - `loadConteneurs()`      : 3 conteneurs actifs

Puis :
```ts
const [quotesRes, commissionsRes, savRes, contRes] = await Promise.allSettled([
  loadQuotesStats(), loadCommissionsStats(), loadSavCount(), loadConteneurs(),
]);
```

Pour chaque section :
- `status === 'fulfilled'` : applique au state
- `status === 'rejected'` : `console.warn` ciblé + state reste à default
  (KPI 0, listes vides → demo data fallback dans le JSX existant)

Bannière `errorMsg` apparaît UNIQUEMENT si TOUTES les sections ont échoué
(mode dégradé extrême — ex: pas de connexion BD du tout).
