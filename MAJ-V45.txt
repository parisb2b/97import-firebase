────────────────────────────────────────────────────────────────────────────
  BUG B — DATE "[object Object]" SUR DEVIS VIP  ✅ RÉSOLU
────────────────────────────────────────────────────────────────────────────

> Fix direct — pas de plan formel (spec V45 : Bug B = "fix direct rapide").

CAUSE RACINE
────────────
Le bloc "N° DVS-XXXX  Date : ..." correspond exactement au header PDF
généré par `pdf-generator.ts:124-125`. Le helper local `formatDate`
(pdf-generator.ts:81-86 avant fix) avait un fallback `String(date)` :

  function formatDate(date: any): string {
    if (!date) return new Date().toLocaleDateString('fr-FR');
    if (date.toDate) return date.toDate().toLocaleDateString('fr-FR');
    if (date instanceof Date) return date.toLocaleDateString('fr-FR');
    return String(date);   // ← BUG : { _seconds, _nanoseconds } → "[object Object]"
  }

Quand un Timestamp Firestore est sérialisé via JSON round-trip (ex : passage
par Cloud Function, signature externe, Admin SDK), il revient sous la forme
`{ _seconds, _nanoseconds }` SANS méthode `.toDate()`. Aucune des branches
ne match → `String({_seconds:...})` retourne la string `"[object Object]"`.

POURQUOI VIP ?
──────────────
Les devis VIP transitent par un flux différent (signature client externe via
URL token, persistance Cloud Function), qui sérialise le `createdAt` en JSON
brut. Les devis non-VIP sont chargés par `getDoc()` natif côté SDK Web →
retourne des `Timestamp` natifs avec `.toDate()` intact. Le bug touchait
donc uniquement la branche VIP, mais le fix bénéficie à TOUS les types de
PDF (devis VIP/normal, factures finales, notes commission, factures
logistiques) — la shape `{_seconds}` peut survenir n'importe où.

FIX APPLIQUÉ
────────────
Délégation au helper `formatDateCourt` de `src/lib/dateHelpers.ts` (V44-BIS).
Ce helper, via `toDate()` interne, gère TOUTES les formes :
  Date | Firestore Timestamp | { _seconds, _nanoseconds } | string ISO | epoch ms

Format affiché : "JJ mois AAAA" — ex : "28 avril 2026" (conforme spec V45).

Egalement appliqué à `signe_le` (date de signature, pdf-generator.ts:467-472).

L'ancien fallback "today's date" pour date manquante a été supprimé (ne pas
mentir sur la date de création) — `formatDateCourt` retourne `'—'` dans ce cas.
