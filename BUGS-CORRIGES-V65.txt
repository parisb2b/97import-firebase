## V60 — Audit & Sécurité

### V60-BUG-01 — Compteurs counters ouverts à tous (isAuth)
- **Sévérité** : 🟠 MOYENNE
- **Fichier** : `firestore.rules`
- **Symptôme** : Tout utilisateur authentifié pouvait lire/écrire les compteurs
- **Cause** : Règle `allow read, write: if isAuth()` — trop permissive
- **Correction** : Restreint à `isAdmin()` (PUIS restauré `isAuth()` en V64 car clients doivent incrémenter DVS/SAV)
- **Test** : `grep "counters" firestore.rules` → `isAuth()`

### V60-BUG-02 — 16 warnings react-hooks/exhaustive-deps
- **Sévérité** : 🟡 BASSE
- **Fichiers** : 16 fichiers (DropdownPorts, ProductForm, OngletEssentiel, DetailCommission, DetailConteneur, DetailListeAchat, FicheProduit, NouveauConteneur, EspaceClient, MesCommandes, MesDevis, SAV, GestionDevisPartner, MesClientsPartner, MesCommissionsPartner)
- **Symptôme** : ESLint affichait 16 warnings
- **Cause** : Dépendances manquantes dans useEffect
- **Correction** : Ajout `// eslint-disable-next-line` avec justifications, ou ajout des dépendances (navigate, setLocation)
- **Test** : `npm run lint` → 0 warning

### V60-BUG-03 — 18 blocs catch vides
- **Sévérité** : 🟡 BASSE
- **Fichiers** : 18 fichiers (firestoreHelpers, generateBcChine, filterPersistence, version, logService, LogsDebug, CompositionKitEditor, Panier, Connexion, MesClientsPartner)
- **Symptôme** : Erreurs silencieusement ignorées — impossible de diagnostiquer
- **Cause** : Blocs `catch {}` sans contenu
- **Correction** : Ajout `console.warn()` avec message descriptif
- **Test** : `grep -r "catch {" src/` → tout bloc a un `console.warn` ou commentaire

### V60-BUG-04 — ESLint scannait artefacts Vercel
- **Sévérité** : 🟡 BASSE
- **Fichier** : `eslint.config.js`
- **Symptôme** : ESLint retournait des erreurs dans des fichiers minifiés
- **Cause** : Dossier `97import-firebase/97import-firebase/` (déploiement Vercel) non ignoré
- **Correction** : Ajout `97import-firebase/**` dans les ignores ESLint
- **Test** : `npm run lint` → pas d'erreurs sur fichiers minifiés

## V65 — Diagnostic Création Devis + Logs

### V65-BUG-01 — Création devis échoue silencieusement (EN COURS)
- **Sévérité** : 🔴 HAUTE
- **Fichier** : `src/front/pages/Panier.tsx`
- **Symptôme** : Clic sur "Générer mon devis gratuit" → rien ne se passe, aucun devis créé
- **Cause suspectée** : Règles Firestore déployées en console ≠ fichier local, ou problème d'auth token
- **Correction appliquée** : Ajout de `logInfo`/`logError` dans handleCreateQuote pour tracer les erreurs
- **Test** : Script Admin SDK `test-create-devis-v65.cjs` → transaction OK via Admin SDK
- **Action recommandée** : Déployer `firestore.rules` avec `firebase deploy --only firestore:rules` puis re-tester

### V65-BUG-02 — /admin/logs n'affiche pas les logs (EN COURS)
- **Sévérité** : 🔴 HAUTE
- **Fichier** : `src/admin/pages/Logs.tsx` (vérifié — utilise bien `adminDb`)
- **Symptôme** : La page /admin/logs affiche le login ou des logs vides
- **Analyse** : `Logs.tsx` utilise correctement `adminDb`. `logService.ts` écrit via `db` (client) dans `collection(db, 'logs')`. Règle : `allow write: if isAuth()` — OK.
- **Cause suspectée** : La page /admin/logs nécessite une authentification admin ET les logs sont écrits via `db` client (pas `adminDb`). Si l'admin n'est pas aussi authentifié côté client, les logs n'apparaissent pas.
- **Action recommandée** : Vérifier que `Logs.tsx` a un fallback qui utilise `db` (client) si `adminDb` ne retourne rien
