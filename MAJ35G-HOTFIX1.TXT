 NOTES TECHNIQUES
================================================================================

COMPRESSION IMAGE:
  - Canvas HTML5 côté client (pas de serveur Node/Sharp)
  - Max width: 1920px (responsive pour écrans Retina)
  - Quality: 0.8 (80%) pour JPEG
  - Format sortie: toujours JPEG (même si PNG/WebP en entrée)
  - Avantages:
    * Pas de quota serveur consommé
    * Upload plus rapide (fichier déjà compressé)
    * Feedback immédiat dans le browser

FIREBASE STORAGE PATH:
  - Image principale: products/{ref}/principale/main_{timestamp}.jpg
  - Galerie (Phase 3): products/{ref}/galerie/{timestamp}_{filename}
  - Documents PDF: products/{ref}/documents/{timestamp}_{filename}
  - Certificat CE: products/{ref}/certifications/CE_{timestamp}.pdf

  Avantages structure:
    * 1 dossier par produit (facile à retrouver)
    * Timestamp évite conflits noms
    * Suppression produit → peut supprimer tout le dossier

SAUVEGARDE PARTIELLE FIRESTORE:
  Structure document product:
  {
    reference: "TEST-001",
    categorie: "solaire",
    nom_fr: "Test",
    // ... autres champs (peuvent être undefined)
    completude: {
      essentiel: 3,        // sur 11
      details: 0,          // sur 20
      medias: 0,           // sur 9
      statut: "bloquant"   // complet | pret_site | a_enrichir | bloquant
    },
    created_at: Timestamp,
    updated_at: Timestamp
  }

VALIDATION RÉFÉRENCE:
  - Seule validation strictement bloquante (avec catégorie)
  - Raison: référence = document ID Firestore
  - Vérification unicité via getDoc() avant création
  - Pas de renommage possible après création (immutable ID)
