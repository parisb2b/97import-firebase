PARTIE 2 — BC-CHINE (✅ COMPLET ET FONCTIONNEL)
══════════════════════════════════════════════════════════════

FICHIER CRÉÉ :
  ✅ src/lib/excel-generators/generateBcChine.ts (130 lignes)

SOURCE DONNÉES :
  Collection Firestore : listes_achat/{laId}
  Enrichissement : products/{ref} pour nom_zh, fournisseur, prix_achat

STRUCTURE EXCEL :
  Ligne 1 : Titre "BC-CHINE 中国采购单 — BON DE COMMANDE CHINE"
  Ligne 2 : Info LA (numéro formaté + date)
  Ligne 4 : En-têtes (N°, Réf, Nom FR, Nom ZH 中文, Qté, Fournisseur, Prix unit ¥, Total ¥)
  Lignes 5+ : Produits (auto-fetch nom_zh + fournisseur depuis products/)
  Ligne TOTAL : fond gris, bordures épaisses, total CNY
  Ligne conversion : taux RMB, équivalence EUR
  Ligne notes : si la.notes présent

GESTION CHAMPS VIDES :
  - nom_zh vide → "À compléter" (italic gris)
  - fournisseur vide → "À compléter" (italic gris)
  - prix_achat 0/null → 0.00

INTÉGRATION UI :
  ✅ Bouton ajouté dans src/admin/pages/DetailListeAchat.tsx
  ✅ Position : après "Enregistrer" et "Envoyer", avant modal
  ✅ Texte : "📊 BC-CHINE (Excel)"
  ✅ Couleur : bleu #1565C0
  ✅ Disabled si lignes.length === 0 ou generatingExcel
  ✅ Auto-fetch taux_rmb depuis admin_params/taux_rmb.valeur (fallback 7.82)
  ✅ Filename : BC-CHINE_{la.numero}_{date}.xlsx

WORKFLOW :
  1. Admin ouvre DetailListeAchat
  2. Ajoute produits via modal (avec kit explosion)
  3. Clique "BC-CHINE (Excel)"
  4. Système :
     - Lit taux RMB
     - Charge LA + lignes
     - Pour chaque ligne : fetch product data
     - Génère Excel avec formatting bilingue
     - Télécharge fichier
  5. Excel s'ouvre : bilingual FR/ZH, prêt pour envoi fournisseur

TESTS MANUELS RÉALISÉS :
  ✅ Build TypeScript : 0 erreur
  ✅ Bundle size : +950KB (exceljs ajouté) → 2.47MB total
  ✅ Import ExcelJS : pas de conflit avec XLSX existant
