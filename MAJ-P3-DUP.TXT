COMPORTEMENT DÉTAILLÉ :
  1. User clique "📋 Dupliquer" sur une fiche existante
  2. Modal s'ouvre avec :
     - Info produit source (ref + nom)
     - Input nouvelle ref (vide, focus auto)
     - Input nouveau nom (pré-rempli "COPIE DE ...")
     - Input catégorie (pré-rempli avec catégorie actuelle)
     - Checkbox "Créer en désactivé" (cochée)
  3. User saisit nouvelle ref + ajuste le nom
  4. Click "Dupliquer" :
     a. Validation formulaire (ref non vide, différente, nom non vide)
     b. Query Firestore where('reference', '==', nouvelleRef)
     c. Si existe → erreur affichée en rouge dans modal
     d. Sinon :
        - Copie tous les champs du produit
        - Supprime : id, groupe_produit, options_config, image_principale
        - Remplace : reference, nom_fr, categorie, actif
        - Ajoute : duplique_de, duplique_le, created_at, updated_at
        - addDoc(collection(db, 'products'), nouveauProduit)
        - Ferme modal
        - Alert "✅ Produit dupliqué..."
        - Redirection vers /admin/produits/{newRef}
  5. User arrive sur la fiche du nouveau produit (vierge de groupe/options/image)
