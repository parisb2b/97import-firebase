Date : 02/05/2026
Branche : v2
Firebase Project ID : importok-6ef77

## 1. Résumé du problème

Après V55 :
- l'espace client affichait le devis DVS-2605006 ;
- l'admin affichait Clients (0) et Tous les devis (0) ;
- Firestore contenait pourtant des données clients et devis.

Objectif V56 :
- rendre visibles les clients et devis dans l'admin ;
- conserver l'accès client à ses devis ;
- nettoyer les doublons/profils liés à interenovations@gmail.com ;
- ne pas supprimer de données protégées ;
- ne pas déployer Firebase Hosting.

## 2. Cause réelle trouvée

Cause principale :
- l'admin utilisait adminAuth, initialisé sur l'app Firebase nommée "admin" ;
- mais les pages admin lisaient Firestore via db, initialisé sur l'app Firebase "client" ;
- les requêtes Firestore admin ne portaient donc pas le token adminAuth attendu par les rules ;
- les pages admin tombaient en erreur de lecture puis affichaient 0.

Correction :
- ajout de adminDb et adminStorage dans src/lib/firebase.ts ;
- bascule des imports Firestore/Storage de src/admin vers adminDb/adminStorage ;
- adaptation des helpers commissionHelpers et quoteStatusHelpers pour utiliser adminDb quand adminAuth.currentUser existe.
