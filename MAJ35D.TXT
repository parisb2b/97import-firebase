FONCTIONNALITÉS CLÉS :

1. LISTE LISTES D'ACHAT (ListeListesAchat.tsx)
   - Filtres statut : TOUS | brouillon | envoyee (boutons style v35)
   - Recherche : numéro, notes, refs produits
   - Tri cliquable 5 colonnes : numero, date_creation, nb_lignes, total_cny, statut
   - useMemo pour filtrage + tri (fix v35c-fix1)
   - Affichage : numero, date, nb produits (badge), total CNY, statut (pill)
   - Clic ligne → navigation vers détail
   - Bouton "+ Nouvelle liste" → page création

2. CRÉATION LISTE (NouvelleListeAchat.tsx)
   - Génération auto numéro LA-YYMMNNN via getNextNumber('LA')
   - Affichage read-only du numéro + statut brouillon
   - Création document vide : lignes=[], total_cny=0, statut=brouillon
   - Redirect automatique vers /admin/listes-achat/{numero}

3. DÉTAIL/ÉDITION LISTE (DetailListeAchat.tsx)
   a) Header :
      - Numéro + badge statut (brouillon orange / envoyee vert)
      - Compteurs : nb produits, total CNY
      - Bouton "Ajouter produits" (si brouillon)
   
   b) Section Informations :
      - Numéro, date création, date envoi (si envoyee)
   
   c) Section Produits (table) :
      - Colonnes : Réf, Nom FR, Qté, Prix unit ¥, Total ¥, Conteneur, Client, Actions
      - Qté éditable (input number) si brouillon
      - Dropdown conteneur (charge conteneurs actifs, exclut livrés)
      - Badge composant kit (🔧 de {kit_parent}) si est_composant_kit
      - Bouton suppression ligne (croix rouge) si brouillon
      - Total CNY calculé via useMemo
   
   d) Section Notes :
      - Textarea éditable si brouillon, read-only sinon
   
   e) Boutons actions :
      - Si brouillon : "Enregistrer (brouillon)" + "Envoyer (figer)"
      - Si envoyee : bandeau "Liste validée — non modifiable"

4. MODAL AJOUT PRODUITS (ModalAjoutProduits)
   a) Chargement devis :
      - Filtre : seulement devis avec acomptes.statut === 'encaisse'
      - Affichage : numero devis, client, nb produits
   
   b) Sélection produits :
      - Checkboxes par ligne de devis
      - Indication "⚠️ Déjà dans cette LA" si ref déjà présente
      - Barre recherche : devis, client, produit
   
   c) Éclatement kits (logique critique) :
      - Lecture produit dans products/{ref}
      - Si est_kit=true ET composition_kit existe :
        → Split composition_kit par ';'
        → Pour chaque composant : lecture products/{compRef}
        → Création ligne par composant avec :
           * ref = compRef
           * nom_fr, fournisseur, prix_achat depuis products
           * qte = qte_devis (pas de multiplication, 1 par kit)
           * kit_parent = ref du kit
           * est_composant_kit = true
      - Sinon : ajout ligne simple
   
   d) Ajout :
      - Traitement async avec loading
      - Compteur sélection dynamique
      - Bouton "Ajouter (X)" désactivé si aucune sélection

5. ENVOI ET MISE À JOUR CONTENEURS
   - Bouton "Envoyer" :
     → Confirmation modale avec avertissement
     → Change statut=envoyee, date_envoi=serverTimestamp()
     → Appelle updateConteneursDevisLies(lignes)
   
   - updateConteneursDevisLies() :
     → Groupe devis_id par conteneur_id
     → Pour chaque conteneur : merge avec devis_lies existants
     → Update conteneurs/{id}.devis_lies = [...existants, ...nouveaux]

6. READ-ONLY MODE
   - Activé si statut === 'envoyee'
   - Désactive :
     * Input qté (affiche valeur)
     * Dropdown conteneur (affiche valeur)
     * Bouton suppression ligne
     * Bouton ajouter produits
     * Textarea notes
     * Boutons enregistrer/envoyer
   - Affiche bandeau gris "Liste validée — non modifiable"
