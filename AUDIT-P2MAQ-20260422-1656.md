### 1.3 Logique de filtrage

```typescript
// Ligne 138 Catalogue.tsx
const catInfo = categorie ? CATEGORIES_INFO[categorie] : null;

// Les produits sont bien filtrés car normalizeCategorie() gère la casse :
const normalizeCategorie = (cat: string | undefined): string => {
  if (!cat) return '';
  return cat.toLowerCase().trim().replace(/\s+/g, '-');
};

const filtered = categorie
  ? all.filter(p => {
      const catNormalisee = normalizeCategorie(p.categorie);
      const catRecherchee = normalizeCategorie(categorie);
      return catNormalisee === catRecherchee && p.actif !== false && p.type !== 'service';
    })
  : all.filter(...);
```

**Analyse** :
- Le filtrage des produits FONCTIONNE car `normalizeCategorie()` normalise la casse
- Mais `catInfo` cherche la clé EXACTE dans `CATEGORIES_INFO` SANS normalisation
- Donc quand categorie='Mini-Pelle', `CATEGORIES_INFO['Mini-Pelle']` → `undefined`

### 1.4 CAUSE EXACTE DU BUG

**BUG #1 : Hero "Catalogue complet" affiché partout**

Quand l'utilisateur clique sur "Mini-Pelles" dans le Header :
1. Header.tsx ligne 99 : `{ path: '/catalogue/Mini-Pelle', ...}` → redirige vers `/catalogue/Mini-Pelle`
2. Catalogue.tsx ligne 49 : `categorie` devient `'Mini-Pelle'` (avec majuscules)
3. Catalogue.tsx ligne 138 : `catInfo = CATEGORIES_INFO['Mini-Pelle']` → `undefined` (clé n'existe pas)
4. Catalogue.tsx ligne 141 : `{catInfo ? (...hero catégorie...) : (...hero complet...)}` → affiche "Catalogue complet"

**BUG #2 : Produits filtrés correctement MAIS hero incorrect**

Les produits sont bien filtrés grâce à `normalizeCategorie()` qui compare :
- URL `Mini-Pelle` normalisé → `mini-pelle`
- Firestore `mini-pelle` normalisé → `mini-pelle`
- Match ✅

Mais le hero ne s'affiche pas car `catInfo` est `undefined`.

**BUG #3 : /catalogue/maison-modulaire ne fonctionne pas**

Même problème : Header link = `/catalogue/Maisons` (≠ `maison-modulaire`)
- Si on tape manuellement `/catalogue/maison-modulaire` dans l'URL, ça devrait fonctionner
- Mais le Header redirige vers `/catalogue/Maisons`, qui ne matche ni CATEGORIES_INFO ni Firestore

## 2. DIAGNOSTIC CONDITION HERO

### 2.1 Code de la condition

```typescript
// Ligne 138
const catInfo = categorie ? CATEGORIES_INFO[categorie] : null;

// Ligne 141
{catInfo ? (
  <section>/* Hero catégorie avec image, couleur, etc. */</section>
) : (
  <section>/* Hero "Catalogue complet" */</section>
)}
```

### 2.2 Pourquoi "catInfo" est null

`catInfo` est `undefined` (pas `null`, mais falsy) car :
- `categorie` = `'Mini-Pelle'` (du Header)
- `CATEGORIES_INFO['Mini-Pelle']` n'existe pas
- Seules les clés `'mini-pelle'`, `'maison-modulaire'`, etc. existent (kebab-case minuscule)

### 2.3 FIX PROPOSÉ

Une seule modification suffit : **corriger les liens Header** (voir section 1.5).

Alternative (moins recommandée) : normaliser `categorie` avant de chercher dans CATEGORIES_INFO :
```typescript
const catKey = categorie.toLowerCase().replace(/\s+/g, '-');
const catInfo = catKey ? CATEGORIES_INFO[catKey] : null;
```
Mais cela masque le vrai problème (Header incorrect).
