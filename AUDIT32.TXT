─── DOMAINE 2 — ACOMPTES (BUG PRINCIPAL) ───
Statut : ❌ CASSÉ

CAUSE RACINE DU BUG DE MICHEL :

Format écrit par le FRONT (PopupAcompte.tsx) :
  acomptes[]: { montant, date, type_compte, statut:'declare', iban_utilise }

Format écrit par l'ADMIN (DetailDevis.tsx handleEncaisser) :
  acomptes[]: { date, montant, ref_fa }
  → PAS DE CHAMP statut !

Filtre "Mes commandes" (MesCommandes.tsx) :
  d.acomptes?.some(a => a.statut === 'encaisse')
  → AUCUN acompte n'a jamais statut='encaisse' → "Mes commandes" est TOUJOURS VIDE

PREUVE dans les données réelles :
  DVS-2604008 acomptes = [{"date":"2026-04-15T09:49:30.688Z","type_compte":"professionnel","statut":"declare","montant":5000}]
  → Le client a déclaré un virement de 5000€, mais l'admin n'a jamais changé le statut
  → Même si l'admin "encaisse", il AJOUTE un NOUVEL objet {date, montant, ref_fa} via arrayUnion
  → L'ancien objet garde statut:'declare' pour toujours

FLUX ATTENDU (qui ne fonctionne PAS) :
  1. Client déclare un virement → acompte ajouté avec statut='declare' ✅
  2. Admin valide l'encaissement → devrait MODIFIER le statut de CET acompte à 'encaisse' ❌
  3. Front filtre sur statut='encaisse' pour afficher dans "Mes commandes" ❌ (jamais encaissé)

FLUX ACTUEL (cassé) :
  1. Client déclare → {montant:5000, statut:'declare', type_compte:'pro', ...}
  2. Admin "encaisse" → AJOUTE un NOUVEL objet {montant:X, date:Y, ref_fa:Z} (sans statut)
  3. On a maintenant 2 entrées dans acomptes[] : l'original 'declare' et le nouveau sans statut
  4. Le front cherche statut='encaisse' → jamais trouvé → "Mes commandes" vide

Recommandations v33 CRITIQUES :
  1. L'admin doit MODIFIER l'acompte existant, pas en ajouter un nouveau
     OU : l'admin doit mettre statut='encaisse' sur l'acompte qu'il valide
  2. Le bouton "Encaisser" dans DetailDevis doit :
     - Lister les acomptes 'declare' du client
     - Permettre de sélectionner lequel encaisser
     - Mettre le statut de CET acompte à 'encaisse'
     - Ajouter le ref_fa à cet acompte
  3. Recalculer total_encaisse depuis la somme des acomptes 'encaisse'
